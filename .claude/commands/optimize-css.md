Analysiere die CSS-Datei `$ARGUMENTS` auf:

1. **Performance**
   - Spezifität-Probleme (zu tiefe Verschachtelung)
   - Ungenutzte Selektoren
   - Redundante Regeln
   - Render-Blocking-Potenzial

2. **Responsive Design**
   - Mobile-First Ansatz
   - Breakpoint-Konsistenz
   - Flexbox vs. Grid Verwendung
   - Viewport-Units

3. **Wartbarkeit**
   - CSS-Variablen (Custom Properties)
   - Konsistente Namenskonvention (BEM, etc.)
   - Logische Gruppierung
   - Kommentare für komplexe Bereiche

4. **Design-System**
   - Farb-Tokens
   - Spacing-System (4px/8px Grid)
   - Typography-Scale

Schlage konkrete Optimierungen vor.
